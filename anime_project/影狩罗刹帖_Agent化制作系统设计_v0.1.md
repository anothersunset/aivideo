# 《影狩罗刹帖》Agent 化制作系统设计 v0.1

## 1. 制作假设

制作商目标：尽可能使用 Agent 完成企划、剧本、美术、分镜、动画、合成、声音、制片管理和技术软件搭建。

核心策略：建立一个多 Agent 动画制作系统，由“导演总控 Agent”和“总制片 Agent”统筹，其他专业 Agent 负责具体生产。人类制作商保留最终绿灯、预算、版权、发行和高风险内容批准权。

原则：
- Agent 负责生成、检查、迭代和归档。
- 所有资产进入统一版本管理。
- 所有镜头按 shot id 流转。
- 所有输出必须经过质量检查 Agent 和导演 Agent 审查。
- 不公开复刻参照作品，只继承成人向忍者动作气质。

## 2. 总体架构

### 三层结构

第一层：决策层
- 制作商：最终批准预算、方向、发行、风险。
- 导演总控 Agent：判断创作方向、镜头风格、表演、节奏。
- 总制片 Agent：控制排期、成本、风险、交付物。

第二层：创作生产层
- 编剧 Agent
- 世界观 Agent
- 角色设计 Agent
- 敌人设计 Agent
- 美术监督 Agent
- 分镜 Agent
- 动作设计 Agent
- Layout Agent
- 原画/动画 Agent
- 摄影合成 Agent
- 声音 Agent
- 宣发 Agent

第三层：技术与质检层
- 管线开发 Agent
- 资产管理 Agent
- 版本控制 Agent
- 版权/原创性检查 Agent
- 尺度/评级检查 Agent
- 镜头连续性检查 Agent
- 画面质量检查 Agent
- 自动报告 Agent

## 3. Agent 职责设计

| Agent | 输入 | 输出 | 审查者 |
|---|---|---|---|
| 导演总控 Agent | 项目圣经、剧本、分镜、样片 | 导演意见、风格判定、修改指令 | 制作商 |
| 总制片 Agent | 排期、预算、任务状态 | 预算报告、风险报告、派工表 | 制作商 |
| 编剧 Agent | 大纲、角色小传、场景表 | 剧本正文、对白修订、场景改写 | 导演 Agent |
| 角色设计 Agent | 小传、视觉关键词 | 角色方向稿提示词、三视图方案 | 导演/美术 Agent |
| 敌人设计 Agent | 百骸八众表 | 剪影方案、能力部件、动作 Pose | 导演/动作 Agent |
| 美术监督 Agent | 视觉参考板、场景表 | 场景设定、色彩脚本、材质规范 | 导演 Agent |
| 分镜 Agent | 剧本、镜头表 | 分镜草图、动态分镜文本、镜头节奏 | 导演 Agent |
| 动作设计 Agent | 敌人能力、场景空间 | 动作拆解、关键 Pose、战斗规则 | 导演 Agent |
| 动画 Agent | 分镜、Layout、角色设定 | 原画草案、中割建议、运动曲线 | 作监/导演 Agent |
| 摄影合成 Agent | 背景、角色层、特效层 | 合成方案、LUT、雨烟火血特效 | 导演 Agent |
| 声音 Agent | 剧本、分镜、节奏点 | 音效表、拟音清单、临时配乐方向 | 导演 Agent |
| 版权检查 Agent | 角色图、文字、宣发 | 相似性风险报告 | 法务/制作商 |
| 尺度检查 Agent | 剧本、分镜、成片 | 导演版/国际版剪辑建议 | 总制片 Agent |

## 4. 需要实现的软件系统

### A. 制作中枢系统

名称建议：Kage Studio Hub

功能：
- 项目文件管理。
- Agent 任务调度。
- 镜头状态看板。
- 版本记录。
- 审查意见流转。
- 自动生成日报/周报。

核心数据对象：
- Project
- Episode/Feature
- Sequence
- Shot
- Character
- Prop
- Background
- Asset
- Review
- Version
- Risk

技术建议：
- 后端：Python FastAPI 或 Node.js。
- 数据库：PostgreSQL。
- 文件索引：SQLite 或 PostgreSQL metadata。
- 前端：Next.js。
- 任务队列：Celery/RQ 或 BullMQ。
- 存储：本地 NAS + 云备份。

### B. 剧本与分镜系统

功能：
- 剧本场景管理。
- 自动拆场。
- 自动生成镜头初稿。
- 镜头编号。
- 分镜文本到画面提示词。
- 动态分镜时间线。

Agent 能力：
- 从剧本提取角色、地点、道具、动作。
- 生成 shot list。
- 生成 storyboard prompt。
- 检查场景连续性。
- 检查对白是否过度解释。

### C. 视觉资产生成系统

功能：
- 角色方向稿生成。
- 三视图辅助生成。
- 表情表生成。
- 道具设计生成。
- 场景概念图生成。
- 色彩脚本生成。

建议流程：
1. 文字设定进入提示词生成 Agent。
2. 视觉生成模型输出多方案。
3. 美术监督 Agent 评分。
4. 导演 Agent 选择方向。
5. 资产管理 Agent 入库。

注意：
- 所有生成结果必须做原创性检查。
- 关键角色不能只依赖一次生成，必须迭代统一。
- 需要建立角色 LoRA/参考一致性方案，或等效角色一致性工具。

### D. 动画生产系统

功能：
- 分镜到 Layout。
- Pose 生成。
- 原画辅助。
- 中割辅助。
- 镜头运动预览。
- 动作曲线检查。

技术路线：
- 2D 为主。
- 关键 Pose 由 Agent 生成多方案。
- 动作设计 Agent 检查节奏。
- 动画 Agent 生成中间帧建议。
- 画面质量 Agent 检查闪烁、变形、角色漂移。

可用技术方向：
- 2D 图像生成模型。
- ControlNet/OpenPose 等姿态控制。
- AnimateDiff 或同类视频生成辅助。
- EbSynth/Runway/ComfyUI 工作流。
- Blender Grease Pencil 或 Toon Boom 等传统工具接口。
- 自研镜头状态和资产管理，不完全依赖单一生成工具。

### E. 摄影合成系统

功能：
- 雨、烟、火、血、毒膜特效合成。
- 色彩 LUT。
- 景深、光效、颗粒。
- 镜头最终合成。

技术路线：
- After Effects/Nuke/Blender/DaVinci Resolve 可由 Agent 调用或生成脚本辅助。
- 对样片重点实现雨夜合成模板。
- 毒膜效果需要单独做 shader 或半透明层工作流。

样片重点模板：
- 雨夜外景模板。
- 温泉蒸汽模板。
- 幻觉香模板。
- 药烟白幕模板。
- 水母尼毒膜显形模板。
- 血入水模板。

### F. 声音系统

功能：
- 自动音效清单。
- 临时配乐生成。
- 拟音提示。
- 声音节奏点匹配。
- 导演版/国际版混音差异管理。

Agent 任务：
- 从分镜提取声音点。
- 标注雨、刀、火、烟、毒膜等音效。
- 生成临时音效参考。
- 检查对白是否被音乐盖住。

### G. 质量与风险检查系统

功能：
- 角色一致性检查。
- 镜头连续性检查。
- 色彩一致性检查。
- 尺度风险检查。
- 原创性相似风险检查。
- 成本超支预警。

核心报告：
- 每日镜头风险报告。
- 每周预算燃烧报告。
- 每版样片审查报告。
- 对外材料版权风险报告。

## 5. Agent 工作流：样片《雨夜温泉宿》

### Step 1：剧本锁定

输入：
- 第一幕正文。
- 样片剧本。
- 镜头表。

执行 Agent：
- 编剧 Agent
- 导演 Agent
- 尺度检查 Agent

输出：
- 样片剧本 v1.0。
- 镜头表 v1.0。
- 风险标记。

### Step 2：分镜生成

执行 Agent：
- 分镜 Agent
- 动作设计 Agent
- 美术监督 Agent

输出：
- 21 镜分镜草图。
- 动态分镜。
- 温泉宿平面图。
- S 级镜头 Pose。

### Step 3：视觉资产

执行 Agent：
- 角色设计 Agent
- 敌人设计 Agent
- 美术监督 Agent
- 版权检查 Agent

输出：
- 凛藏、鸢、水母尼方向稿。
- 温泉宿场景设定。
- 毒膜视觉测试。
- 原创性风险报告。

### Step 4：动画与合成

执行 Agent：
- Layout Agent
- 动画 Agent
- 摄影合成 Agent
- 质量检查 Agent

输出：
- S 级镜头动画测试。
- 雨烟火血毒膜合成测试。
- 样片画面 v0.1。

### Step 5：声音与终版

执行 Agent：
- 声音 Agent
- 剪辑 Agent
- 导演 Agent

输出：
- 样片 v1.0。
- 导演版。
- 投资展示版。
- 成本复盘报告。

## 6. 自研软件优先级

### 第一优先级：必须做

1. 项目中枢与镜头看板。
2. 资产命名和版本管理。
3. Agent 任务调度。
4. 剧本到镜头表转换。
5. 审查意见系统。

### 第二优先级：样片前必须做

1. 分镜生成辅助。
2. 角色一致性管理。
3. 视觉提示词管理。
4. 摄影特效模板。
5. 样片预算追踪。

### 第三优先级：长片制作前必须做

1. 自动动态分镜工具。
2. 动作 Pose 库。
3. 镜头连续性检查。
4. 版权相似性检查。
5. 国际版剪辑差异管理。

## 7. 技术栈建议

### 后端

- Python FastAPI：适合 AI 工作流和媒体处理。
- PostgreSQL：项目和镜头数据。
- Redis：任务队列缓存。
- Celery/RQ：长任务调度。

### 前端

- Next.js：制作看板、审查界面、资产浏览。
- Tailwind 或等效 UI 系统。

### AI 工作流

- ComfyUI：图像生成与节点式视觉流程。
- Stable Diffusion/Flux 等图像模型：概念图、角色方向稿。
- 视频生成模型：动作测试，不作为最终唯一来源。
- LLM Agent：剧本、拆镜、审查、报告。
- OCR/视觉模型：检查图像文字、构图、角色一致性。

### 媒体处理

- FFmpeg：转码、代理文件、时间码。
- Blender：3D/Grease Pencil/镜头预演。
- DaVinci Resolve/After Effects/Nuke：后期合成，可逐步脚本化。

### 版本管理

- Git：文本、代码、配置。
- DVC 或类似方案：大文件资产版本。
- Shot id 文件夹规范：所有镜头资产归档。

## 9. 人类审批点

即使全部使用 Agent 制作，以下环节必须由制作商或指定人类负责人批准：

1. 项目定位和预算。
2. 对外融资材料。
3. 角色最终设计。
4. 重大剧情修改。
5. 高风险暴力/尺度镜头。
6. 法务原创性风险。
7. 样片最终版。
8. 长片绿灯。
9. 平台/发行合同。
